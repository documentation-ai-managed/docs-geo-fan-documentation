## Overview

GEO.fan empowers you to create, organize, and manage your project documentation seamlessly. You get a centralized space to build comprehensive guides, API references, and tutorials using powerful MDX components. Whether you document software projects, internal wikis, or product handbooks, GEO.fan streamlines the process with intuitive tools and rich formatting options.

## What is GEO.fan?

GEO.fan is your default documentation hub. You can structure content hierarchically, embed interactive elements like tabs and steps, and ensure consistent branding across all pages. It supports advanced MDX syntax for dynamic components, making your docs engaging and professional.

<Callout kind="info">
  GEO.fan uses brand color `#3B82F6` for primary elements. Apply it consistently in custom themes or CSS.
</Callout>

## Get Started in Minutes

Follow these steps to set up your first documentation site.

<Steps>
  <Step title="Create Account" icon="user-plus">
    Sign up at `https://geo.fan` and verify your email.
  </Step>
  <Step title="New Project" icon="plus">
    Click "New Space" and name it, e.g., "My Project Docs".
  </Step>
  <Step title="Add Pages" icon="file-text">
    Create `introduction.mdx` and start writing with MDX syntax.
  </Step>
  <Step title="Publish" icon="globe">
    Hit publish to share your live documentation site.
  </Step>
</Steps>

## Multi-Language Integration Examples

Integrate GEO.fan APIs across languages.

<CodeGroup tabs="JavaScript,Python,cURL">
  ```javascript
  const response = await fetch('https://api.geo.fan/v1/docs', {
    headers: { 'Authorization': 'Bearer YOUR_API_KEY' }
  });
  const docs = await response.json();
  console.log(docs);
  ```
  ```python
  import requests
  response = requests.get(
      'https://api.geo.fan/v1/docs',
      headers={'Authorization': 'Bearer YOUR_API_KEY'}
  )
  docs = response.json()
  print(docs)
  ```
  ```bash
  curl -H "Authorization: Bearer YOUR_API_KEY" \
       https://api.geo.fan/v1/docs
  ```
</CodeGroup>

## Brand Guidelines

<Expandable title="Detailed Brand Guidelines" default-open="false">

Adhere to these rules for consistency:

- **Primary Color:** `#3B82F6` (use for buttons, links, headers)
- **Typography:** Sans-serif fonts, 16px base size
- **Icons:** Lucide library with blue tint matching `#3B82F6`

| Element       | Color       | Usage                  |
|---------------|-------------|------------------------|
| Primary Button| `#3B82F6`  | Actions and CTAs      |
| Text          | `#1F2937`  | Body and headings     |
| Background    | `#FFFFFF`  | Main content areas    |

Avoid overuse of secondary colors to maintain focus.

</Expandable>

This setup ensures your documentation remains professional and on-brand. Start building today!
