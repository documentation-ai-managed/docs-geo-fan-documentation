## Overview

Get up and running with GEO.fan in minutes. This guide walks you through creating an account, setting up your first project, navigating the dashboard, and creating your initial document. GEO.fan provides a powerful space to organize project documentation with MDX support, version control, and collaborative features.

<Callout kind="tip">
  Use your brand color `#3B82F6` in custom themes for a cohesive look.
</Callout>

## Quickstart Steps

Follow these steps to launch your first documentation space.

<Steps>
  <Step title="Create Account" icon="user-plus">
    Visit [geo.fan](https://geo.fan) and click **Sign Up**.

    Fill in your email, password, and team name. Verify your email to activate the account.
  </Step>

  <Step title="Set Up Project" icon="plus">
    From the dashboard, select **New Project**.

    Enter project details:

    ```
    Name: My Project Docs
    Description: Organized docs for my app
    Slug: my-project-docs
    ```

    Choose public or private visibility.
  </Step>

  <Step title="Navigate Dashboard" icon="layout">
    Access key sections:

    - **Documents**: Manage pages and folders
    - **Settings**: Configure themes and integrations
    - **Analytics**: View visitor stats
    - **Members**: Invite collaborators
  </Step>

  <Step title="Create Document" icon="file-text">
    Click **New Document** > **Blank Page**.

    Add frontmatter and content. Publish to make it live.
  </Step>
</Steps>

## Create Your First Document

Start with a basic MDX page. Use the editor or import from GitHub.

<Tabs>
  <Tab title="Web Editor" icon="edit-3">
    Paste this starter template:

    ```mdx
    ---
    title: Welcome
    description: Your project homepage.
    ---

    ## Introduction

    Welcome to your GEO.fan docs.
    ```
  </Tab>

  <Tab title="GitHub Import" icon="github">
    Connect your repo and sync automatically.
  </Tab>
</Tabs>

<CodeGroup tabs="MDX,YAML">
  ```mdx
  ---
  title: API Reference
  ---

  ## Endpoints

  Use `{API_KEY}` for auth.
  ```
  ```yaml
  title: API Reference
  description: Endpoints and params
  sidebar:
    - /api/intro
    - /api/users
  ```
</CodeGroup>
